# Darim (Отдам даром)

Мобильное приложение для бесплатного обмена вещами. Проект помогает людям отдавать хорошие, но ненужные вещи тем, кто в них нуждается, решая проблему захламления квартир и неэкологичного выбрасывания.

---

 Стек технологий

| Компонент | Технология |
|-----------|------------|
| Язык | Kotlin |
| Минимальная версия Android | API 24 (Android 7.0) |
| Целевая версия Android | API 34 (Android 14) |
| UI | XML layouts, Material Design |
| Навигация | Fragment Manager + BottomNavigationView |
| Асинхронность | Coroutines + Flow |
| Геолокация | Google Play Services Location |
| Архитектура | Clean Architecture (3 слоя: domain, data, presentation) |
| Состояние | LiveData + ViewModel |
| Сборка | Gradle Kotlin DSL |

## Как запустить
1. Клонировать репозиторий git clone https://github.com/Malinovskiy159/mobile_app.git
2. Открыть в Android Studio
Запустите Android Studio
Выберите File → Open...
Укажите путь к скачанной папке mobile_app
Дождитесь загрузки проекта
3. Синхронизация Gradle
После открытия Android Studio автоматически начнет синхронизацию Gradle
Дождитесь сообщения "Gradle sync finished" в нижней панели
Если синхронизация не началась: File → Sync Project with Gradle Files
4. Запуск приложения
Создайте эмулятор с Android 7.0+ (API 24)
В верхней панели инструментов выберите конфигурацию app
Нажмите зеленую кнопку Run или используйте Shift + F10
Дождитесь сборки и установки приложения
5. Важная настройка фильтра
После запуска приложения обязательно:
Откройте экран фильтрации
Выберите опцию "Поиск во всем городе"
Почему это важно:
Эмулятор по умолчанию определяет геолокацию в Калифорнии (США)
Если радиус поиска ограничен, вы не увидите объявления из вашего региона
Установка "поиска по всему городу/миру" отключает ограничение по радиусу



##  Моки
Заглушки данных находятся в `data/repository/ItemRepositoryImpl.kt` — список `allItems` с тестовыми объявлениями (ID, название, категория, описание, координаты, телефон).

##  Ключевой сценарий: "Найти вещь на карте и забронировать"

### Описание сценария
Пользователь (забирающий) открывает главный экран со списком объявлений, находит нужную вещь, просматривает детали и бронирует её. После подтверждения открываются 
контактные данные продавца.

### Шаги для проверки

#### 1. Запустите приложение
- Должен открыться главный экран с сеткой объявлений (2 колонки)

#### 2. Проверьте фильтрацию
- (обязательно в фильтре укажите искать по всему городу,а то геолокация определяется в калифорнии,а радиус поиска ограничен)
- Нажмите на иконку фильтра
- Выберите категорию "Техника"
- Нажмите "Применить"

#### 3. Проверьте геолокацию
- При первом запуске приложение запросит разрешение на геолокацию
- Разрешите доступ
- В карточках должно отображаться расстояние 

#### 4. Проверьте детали объявления
- Нажмите на любое объявление
- Откроется экран с фото (заглушка), названием, описанием и контактами

#### 5. Проверьте бронирование
- Нажмите кнопку "Заберу"
- Подтвердите в диалоговом окне
- Должен открыться диалог с телефоном продавца
- Статус вещи должен измениться на "Забронировано"

#### 6. Проверьте навигацию
- Нажмите кнопку "Назад" — вернетесь на главный экран
- Фильтры должны сохраниться
- Забронированная вещь должна исчезнуть из списка

#### 7. Проверьте профиль
- Нажмите на иконку профиля (пятая внизу)
- Попробуйте добавить фото (клик на аватар)
- Проверьте автоопределение геолокации
- Введите адрес вручную(пока в разработке)

## Известные особенности
- 
- Все данные хранятся в памяти (заглушки), при перезапуске изменения сбрасываются
- Фото — иконки-заглушки
